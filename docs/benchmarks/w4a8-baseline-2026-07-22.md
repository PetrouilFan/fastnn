# W4A8 CPU microkernel baseline — 2026-07-22

## Scope

This is a local operator-level baseline, not an end-to-end promotion result. It compares the existing packed I4 weight paths for MatMul:

- `w4a32_unpack_float`: F32 activations through `gemm_cpu_flat::<I4x8>`.
- `w4a8_direct_i32`: pre-quantized I8 activations through `gemm_cpu_flat_i8_i4x8`.
- `f32_native`: native F32 weights and activations through `gemm_cpu_flat::<F32x1>`.

Activation quantization is performed before the timed region except in the explicitly named combined path. Weight packing is outside the timed region. All paths write F32 outputs. A controlled comparison found the existing AVX2 nibble-expansion implementation to be about 7x slower than the compiler-optimized scalar loop on this Zen 2 host, so W4A8 now selects the portable scalar path while retaining the AVX2 implementation for future iteration.

## Environment

- CPU: AMD Ryzen 7 3700X, 8 cores / 16 threads.
- ISA available: AVX2; no AVX-512.
- OS: Linux 7.1.3-arch1-1 x86_64.
- Build: Criterion release benchmark.
- Command:

```bash
cargo bench --bench cpu_baselines -- w4_activation_paths --sample-size 20 --measurement-time 1
```

## Results

| Workload | W4A32 float path | W4A8 integer path | Relative speedup |
|---|---:|---:|---:|
| Decode `M=1, K=N=256` | 8.8233 ms | 67.705 µs | 130.32x |
| Prefill `M=8, K=N=256` | 18.471 ms | 525.45 µs | 35.15x |

The native F32 packed baseline was measured separately:

| Workload | Native F32 | W4A8 including activation quantization | Relative speedup |
|---|---:|---:|---:|
| Decode `M=1, K=N=256` | 8.4904 ms | 65.037 µs | 130.55x |
| Prefill `M=8, K=N=256` | 18.594 ms | 533.36 µs | 34.86x |

The native F32 and W4A32 paths have similar performance at these shapes. This confirms that the large W4A8 gap is not solely caused by packed-I4 float decoding, but it does not establish competitiveness against an external BLAS or a future blocked F32/BF16 kernel.

An additional timed path includes F32-to-I8 activation quantization into a reused payload buffer:

| Workload | W4A8 including activation quantization | Speedup over W4A32 float |
|---|---:|---:|
| Decode `M=1, K=N=256` | 65.037 µs | 135.67x |
| Prefill `M=8, K=N=256` | 533.36 µs | 34.63x |

Activation quantization remains small relative to the integer GEMM at these shapes. Separate Criterion runs and scheduling outliers make sub-5% deltas non-actionable.

Forced-path comparison:

| Workload | Rejected AVX2 path | Scalar reference | AVX2 slowdown |
|---|---:|---:|---:|
| Decode `M=1, K=N=256` | 450.90 µs | 64.475 µs | 6.99x |
| Prefill `M=8, K=N=256` | 3.5275 ms | 514.03 µs | 6.86x |

Criterion ranges:

- Decode float: 8.6163–9.1084 ms.
- Decode selected W4A8: 64.307–70.855 µs.
- Prefill float: 18.223–18.707 ms.
- Prefill selected W4A8: 511.34–542.56 µs.
- Decode W4A8 including quantization: 64.778–65.378 µs.
- Prefill W4A8 including quantization: 514.80–551.26 µs.
- Decode native F32: 8.4149–8.5487 ms.
- Prefill native F32: 18.461–18.728 ms.

Criterion reported outliers in each sample set, so these numbers are a baseline rather than a stable release claim. The large gap also indicates that the current float packed path is a poor comparison baseline and must be compared against optimized wider kernels before promotion.

## Storage accounting for `N=K=256`

- Packed I4 payload: 32,768 bytes.
- Per-row F32 scales and offsets: 2,048 bytes.
- Effective packed weight bytes before allocator/alignment overhead: 34,816 bytes.
- F32 weight bytes: 262,144 bytes.
- Effective W4 metadata-inclusive ratio: 13.28% of F32.
- Decode I8 activation payload including 8-byte affine header: 264 bytes versus 1,024 F32 bytes.
- Prefill I8 activation payload including header: 2,056 bytes versus 8,192 F32 bytes.

The packed executable layout may additionally reserve SIMD suffix bytes. Prepared-plan and arena-copy accounting must therefore be measured separately.

## Correctness evidence

The existing integration tests passed against this implementation:

```text
test_matmul_u4_i8_dispatch_path
 test_matmul_i4codebook_roundtrip
```

The quantized pipeline suite also passed before this benchmark was added. This benchmark itself does not establish model-level quality.

## Promotion gaps

The W4A8 path is not yet promoted as the production baseline. Remaining evidence:

1. Compare against optimized F32, BF16/F16, and unpack-to-I8 implementations rather than only the current packed-float fallback.
2. Redesign the rejected AVX2 kernel before reconsidering it; evaluate AVX-512/NEON kernels on supported machines.
3. Include activation quantization cost in operator and end-to-end measurements.
4. Measure allocator copies, scratch bytes, prepared-plan behavior, and persistent packed-weight reuse.
5. Add larger decode and prefill shapes representative of supported models.
6. Run deterministic YOLO output parity and COCO smoke mAP.
7. Record repeated runs with controlled CPU governor and reduced scheduling noise.
