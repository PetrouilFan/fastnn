# v2.3 Roadmap

This document tracks the remaining work for fastnn v2.3 after the post-v2.2.4 stabilization sprint. The immediate v2.3 focus is now CPU performance and CPU backend maintainability.

**Current stable release: v2.2.4**

## Completed after v2.2.4

These items were previously listed as planned v2.3 work and are now implemented on `main`:

- **macOS release-test restoration**
  - Fixed macOS aarch64 test instability by zero-initializing CPU arena allocations.
  - Restored macOS runner coverage in `.github/workflows/release.yml`.
- **WGPU pooling and shader/pipeline caching**
  - Added bucketed transient WGPU storage-buffer pooling.
  - Added shader and compute-pipeline cache paths for repeated WGPU dispatches.
  - Added cache telemetry tests to catch hit/miss accounting regressions.
- **Optimizer host-sync reduction**
  - Reduced unnecessary host synchronization in compiled-training optimizer paths.
  - Remaining work is now a CPU/compiler audit and benchmark item rather than a blank implementation item.
- **Release hygiene**
  - Added/updated release preflight checks.
  - Removed stray root-level utility/test files that were not part of the maintained public suite.

## P0 — CPU benchmark expansion

The maintained `cpu_baselines` suite is the release gate for CPU performance work. It should cover more than packed GEMV/GEMM before optimization claims are made.

**Scope:**
- Add `cpu_elementwise` benchmarks for same-shape, scalar, and broadcasted add/mul/div style workloads.
- Add `cpu_reductions` benchmarks for 1D and row-wise sum/mean/max workloads.
- Add `cpu_fusions` benchmarks for matmul+bias+activation and residual+add+norm workloads.
- Add `cpu_training_updates` benchmarks for optimizer update loops where stable APIs allow it.
- Export Criterion results to normalized JSON for release review.

**Status:** In progress.

## P1 — CPU copy/allocation telemetry

The CPU backend contains several arena-to-temporary copy paths. Before changing hot code, those copies should be measurable.

**Scope:**
- Add counters for arena temporary copies and copied bytes.
- Add counters for TLS vector-pool allocation/reuse.
- Keep counters low-overhead and testable.
- Use telemetry to verify later copy-reduction work.

**Status:** In progress.

## P1 — CPU arena slice copy reduction

Reduce avoidable CPU memory traffic in common dispatch paths.

**Scope:**
- Add borrow-safe arena slice helpers with overlap checks and copy fallback.
- Convert scalar op dispatch to read scalar operands directly instead of copying tiny vectors.
- Convert same-shape elementwise binary dispatch to borrowed slices where input/output ranges are disjoint.
- Preserve copy fallback for aliasing and broadcast cases.
- Validate with CPU oracle tests and telemetry counter changes.

**Status:** Not started. Depends on CPU telemetry landing first.

## P1 — CPU matmul and fused epilogue improvements

Improve transformer/MLP-style CPU workloads after benchmark coverage exists.

**Scope:**
- Compare direct microkernel benchmarks against full AOT `CpuBackend` dispatch overhead.
- Tune BLAS vs internal microkernel thresholds if evidence supports it.
- Fuse bias and activation into the internal matmul output path where safe.
- Validate ReLU/GELU/SiLU tolerances and avoid unsupported public speed claims.

**Status:** Not started. Depends on CPU benchmark expansion.

## P1 — CPU reductions and normalization

Improve reduction-heavy CPU paths used by layer norm, RMS norm, and residual+add+norm workloads.

**Scope:**
- Benchmark row-wise reductions at transformer hidden sizes.
- Tune Rayon thresholds for small vs large tensors.
- Avoid repeated passes and repeated bounds checks where safe.
- Preserve numerical stability for mean/variance.

**Status:** Not started. Depends on CPU benchmark expansion.

## P2 — CPU backend module split

`src/backend/cpu/mod.rs` is very large and difficult to review. Split it after performance-sensitive changes are measured and landed.

**Scope:**
- Move arena helpers into `src/backend/cpu/arena.rs`.
- Move telemetry into `src/backend/cpu/telemetry.rs`.
- Move elementwise/scalar dispatch helpers into `src/backend/cpu/elementwise.rs`.
- Move matmul/fusion helpers into `src/backend/cpu/matmul.rs`.
- Move normalization helpers into `src/backend/cpu/norm.rs`.
- Keep `mod.rs` as routing and trait implementation.

**Status:** Deferred until after P1 work to avoid refactor/optimization conflicts.

## Deferred / future

These remain useful but should not displace the CPU-first v2.3 work:

- WGPU-resident arena execution.
- Maintained WGPU benchmark baseline.
- Multi-GPU training with real device-resident gradient synchronization.
- 2-bit quantization.
- Sparse computation.
- Process-based multiprocessing for Python data loading.
- Fastnn-native training graph export metadata.

## Dependency graph

```text
P0 CPU benchmarks          ────> CPU perf claims and optimization targets
P1 CPU telemetry           ────> copy-reduction verification
P1 arena copy reduction    ────> lower dispatch overhead
P1 matmul/fusion work      ────> MLP/transformer CPU speedups
P1 reductions/norms        ────> transformer CPU speedups
P2 CPU module split        ────> maintainability after behavior stabilizes
```

## Release recommendation

- Use **v2.2.5** for benchmark/docs/telemetry-only work.
- Use **v2.3.0** once benchmark expansion plus at least one measured CPU hot-path improvement lands with reproducible artifacts.
