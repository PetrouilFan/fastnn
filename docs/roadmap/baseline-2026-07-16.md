## YOLO11n output-parity and latency baseline

Command:

```bash
.venv/bin/python scripts/yolo_compare_fastnn_pytorch.py \
  --models yolo11n --dtype f32 \
  --pt yolo11n.pt --onnx yolo11n.onnx \
  --imgsz 640 --warmup 3 --iters 10 --profile \
  --profile-json .hermes/baselines/yolo11n-f32-prepared-fix.json
```

| Metric | Result |
|---|---:|
| Output shape | `[1, 84, 8400]` |
| Maximum absolute error vs PyTorch | `0.005737` |
| Mean absolute error vs PyTorch | `0.000002` |
| NaN / Inf | `false / false` |
| fastnn mean latency | `96.21 ms` |
| fastnn observed range | `91.45–101.19 ms` |
| ONNX Runtime mean latency | `28.13 ms` |

The dominant fastnn costs were Conv2d+SiLU (`58.91 ms`) and transpose (`22.30 ms`). This is a correctness baseline, not an optimization claim; fastnn remains substantially slower than ONNX Runtime on this workload.

## Baseline defects found and fixed

Phase 0 exposed and repaired:

1. Quantized wire payload length being conflated with aligned physical slot capacity.
2. Static row-packed and runtime self-describing quantized payloads sharing an ambiguous dequantization mode.
3. Runtime Quantize omitting the explicit no-cached-metadata flag.
4. Prepared Conv2d validation rejecting a plan after runtime tightening appended resolved dimensions.
5. Stale Python importer tests asserting retired operation aliases.
6. A stale Python package version (`2.4.0`) inconsistent with Cargo/project version `2.5.0`.
